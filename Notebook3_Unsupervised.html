Notebook 3: Unsupervised Learning (PCA and k-means)

In this notebook we follow the unsupervised learning ideas from the lecture notes:

  • Dimensionality reduction with PCA to project songs into a low-dimensional space;
  • k-means clustering to find groups of similar songs in that space.

We work only with numerical audio features (plus duration_min and release_year) and keep the code simple and close to what was shown in class.

1. Load the modeling dataset

We start from spotify_model_df.csv, created in Notebook 2. This dataset already has basic cleaning and feature engineering (for example duration_min and release_year).

2. Build the feature matrix from numerical audio features

We select only the numerical audio features plus the engineered features duration_min and release_year. We do not include categorical variables here, to keep the PCA interpretation simple.

If there are still any missing values, we replace them with the median of the corresponding column.

4. PCA: how many components are important?

We fit PCA using all components and then look at the explained variance ratio of each principal component. This tells us how much of the total variability in the data is captured by each component.

We will then focus on the first two principal components (PC1 and PC2) for visualization.

In[4]:
pca = PCA(n_components=None, random_state=0)
pca.fit(X_scaled)

explained = pca.explained_variance_ratio_
explained

plt.ylabel('Cumulative explained variance')
plt.title('Cumulative variance explained by PCA')
plt.ylim(0, 1.05)
plt.show()

Interpretation

The first principal component (PC1) explains a substantially larger share of the total variation in the audio features (around 35%). This means that PC1 captures the dominant underlying pattern across the dataset, a single direction in feature space that accounts for the largest differences among all songs.

The second component (PC2) explains a much smaller but still meaningful proportion of variance (about 14%), and the contribution of further components decreases smoothly.

For the purposes of visualization, this plot justifies why projecting the data onto the first two principal components is reasonable: together they capture roughly 50% of the total information contained in the standardized numerical audio features. This level of variance explained is typical for high-dimensional human-generated data such as musical attributes, where no single feature dominates completely.

Importantly, PCA does not tell us which songs are popular, it simply reduces dimensionality by capturing the strongest directions of variation across musical attributes.

The cumulative curve shows how much of the total variability in the audio features is captured as we include more principal components.

The first two components capture nearly 50% of the variance; using four components gets us to almost 70%; using all components naturally reaches 100%.

This indicates that the audio feature space is not low-dimensional: many different musical attributes contribute meaningfully to how songs differ from one another (e.g., danceability, energy, loudness, tempo, acousticness). Therefore, PCA is useful mainly for visualization and exploration, rather than for compressing the dataset into a very small number of components.

The curve also shows no clear elbow, meaning there is no natural number of principal components that compresses most of the information into a small subspace. This is consistent with the idea that musical characteristics vary along multiple independent axes.

5. Project data onto the first two principal components

To visualize the songs in 2D, we project each standardized example onto the first two principal components. We then make a scatter plot of PC1 vs PC2. If track_popularity is available, we color the points by their true popularity score, so we can see whether more popular songs concentrate in certain regions.

In[6]:
X_pca = pca.transform(X_scaled)  # shape (n_samples, n_features)

pc1 = X_pca[:, 0]
pc2 = X_pca[:, 1]

plt.figure(figsize=(8, 6))

plt.xlabel('PC1')
plt.ylabel('PC2')
plt.title('Songs projected on the first two principal components')
plt.show()

Interpretation of the PCA scatterplot

This scatterplot projects every song onto the first two principal components (PC1 and PC2). Each point represents a song, and its color indicates the true Spotify popularity score.

A few observations emerge:

  1. There is no sharp visual separation between popular and less popular songs.
    Popularity is spread throughout the PCA space, which confirms that popularity is not driven by one or two simple combinations of audio features.

  2. Some mild structure is visible. The slightly lighter/greener points (representing higher popularity) appear somewhat more concentrated on the right-hand side of the plot.
    This suggests that PC1 might correlate weakly with characteristics that tend to make songs more popular for example, higher danceability, energy, or loudness but the effect is not strong.

  3. Popular songs are not isolated or clustered. This is consistent with the fact that popularity depends on many external factors (marketing, artist reputation, playlist placement, genre trends) that PCA does not capture.

Overall, PCA reveals that the audio features do contain some structure, but popularity is not something that emerges naturally from an unsupervised projection of the data. This justifies the use of supervised learning later in the project.

6. k-means clustering in PCA space

Now we apply k-means clustering to group songs in the 2D PCA space. We choose a small number of clusters, for example k = 3, to keep the visualization clear.

The algorithm finds three centroids and assigns each song to the nearest centroid.

In[7]:
k = 3  # number of clusters (you can try 2, 3, 4, ...)
kmeans = KMeans(n_clusters=k, n_init=10, random_state=0)

# We cluster on the first two PCs
X_pca_2d = np.column_stack((pc1, pc2))
kmeans.fit(X_pca_2d)
cluster_labels = kmeans.labels_

kmeans.cluster_centers_

Interpretation of the k-means clustering results

The k-means algorithm identifies three clusters based solely on audio features projected into the first two PCA components.

Important things:

  1. The clusters reflect musical similarity, not popularity.
    Since k-means does not use popularity during training, the resulting groups represent natural groupings of songs with similar audio profiles.
    For example, one cluster may contain more energetic and loud tracks, while another contains softer or more acoustic songs.

  2. The clusters form broad, continuous regions rather than tight and well-separated groups.
    This suggests that musical attributes change gradually across songs rather than forming clear, discrete song types.

  3. There is no clear popularity-based separation.
    When comparing this plot with the popularity-colored PCA scatterplot, we see that popular and less popular tracks are mixed within each cluster.
    Popularity does not correspond to a specific cluster, confirming again that audio features alone do not create distinct popularity categories.

  4. What we gain from clustering is an understanding of the structure* of musical diversity in the dataset, not an explanation of popularity itself.
    Clustering helps us describe the dataset, but predicting and explaining popularity requires supervised learning.

Thus, k-means is valuable for exploratory analysis, but it cannot uncover the ingredients of a perfect song by itself.
